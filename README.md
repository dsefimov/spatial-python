# Документация для контрибьюторов учебного курса

Этот репозиторий содержит исходные материалы курса «Введение в Python: основы синтаксиса». Данная документация предназначена для разработчиков и преподавателей, которые хотят вносить изменения или дополнять материалы.

## Вклад в проект

Если вы хотите внести изменения:
1. Создайте форк репозитория и рабочую ветку
2. Внесите изменения и проверьте сборку
3. Создайте Pull Request с описанием изменений

## Публикация для учащихся

Материалы автоматически публикуются через GitHub Pages. Все ноутбуки содержат кнопки для открытия в Google Colab, что позволяет учащимся работать без локальной установки.

### Для контрибьюторов
- Убедитесь, что ноутбуки выполняются в Google Colab
- Избегайте зависимостей, недоступных в Colab по умолчанию
- Проверяйте, что код не требует специальных настроек

## Сборка и просмотр книги

### Сборка
```bash
# Полная сборка с выполнением ноутбуков
jupyter-book build --all materials
```

### Просмотр

Найдите `index.html` в `materials/_build/html` и откройте его в браузре.  

Либо запустите локальный сервер:
```bash
# Запуск локального сервера
python -m http.server -d materials/_build/html
# Откройте http://localhost:8000
```

## Внесение изменений

### Добавление новой главы
1. Создайте директорию в `materials/content/` (например, `chapter6/`)
2. Добавьте файлы контента (`.md` или `.ipynb`)
3. Обновите `materials/_toc.yml`:

```yaml
chapters:
  # ... существующие главы ...
- file: content/chapter6/intro.md
  sections:
  - file: content/chapter6/26.new_topic.ipynb
```

### Добавление раздела в существующую главу
Добавьте файл в соответствующую секцию `_toc.yml`:

```yaml
- file: content/chapter1/intro.md
  sections:
  - file: content/chapter1/1.basics_of_coding.ipynb
  - file: content/chapter1/new_section.ipynb    # ← новый файл
```

### Изменение существующих материалов
- Редактируйте файлы в соответствующих директориях
- После изменений пересоберите книгу

### Обновление конфигурации
- `materials/_config.yml` — настройки книги
- `pyproject.toml` — зависимости проекта

## Правила именования файлов
- **Директории глав**: `chapter1`, `chapter2`, ..., `practice`, `exams`
- **Введение в главу**: `intro.md`
- **Учебные материалы**: `<номер>.<описание>.<расширение>` (например, `1.basics_of_coding.ipynb`)
- **Практические задания**: `<номер>.practice.ipynb`
- **Тесты**: `<номер>.<тип>.ipynb` (`midterm`, `final`)
