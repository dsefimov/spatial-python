# Документация для контрибьюторов учебных материалов

Этот репозиторий содержит материалы «Введение в Python: основы синтаксиса». Данная документация предназначена для разработчиков и преподавателей, которые хотят вносить изменения или дополнять материалы.

## Нашли ошибку или есть предложение

Если вы не планируете вносить правки самостоятельно, просто создайте [Issue](https://github.com/dsefimov/spatial-python/issues) с описанием проблемы или идеи. Мы обязательно рассмотрим.

## Как внести изменения самостоятельно

### 1. Общий порядок действий

1. Создайте форк репозитория на GitHub
2. Создайте рабочую ветку от `main` (формат: `feature/описание` или `fix/описание`)
3. Внесите изменения и проверьте сборку книги
4. Отправьте изменения в свой форк
5. Создайте Pull Request из вашей ветки в `main` основного репозитория

### 2. Что и как менять

**Добавление новой главы**
1. Создайте директорию в `materials/content/` (например, `chapter6/`)
2. Добавьте файлы введения в главу и содержащие контент (`.md` или `.ipynb`)
3. Обновите `materials/_toc.yml`:

```yaml
chapters:
  # ... существующие главы ...
- file: content/chapter6/intro.md
  sections:
  - file: content/chapter6/26.new_topic.ipynb
```

**Добавление раздела в существующую главу**
Добавьте файл в соответствующую секцию `_toc.yml`:

```yaml
- file: content/chapter1/intro.md
  sections:
  - file: content/chapter1/1.basics_of_coding.ipynb
  - file: content/chapter1/new_section.ipynb    # ← новый файл
```

**Изменение существующих материалов**
- Редактируйте файлы в соответствующих директориях
- После изменений пересоберите книгу

**Обновление конфигурации**
- `materials/_config.yml` — настройки книги
- `pyproject.toml` — зависимости проекта

**Правила именования файлов**
- **Директории глав**: `chapter1`, `chapter2`, ..., `practice`, `exams`
- **Введение в главу**: `intro.md`
- **Учебные материалы**: `<номер>.<описание>.<расширение>` (например, `1.basics_of_coding.ipynb`)
- **Практические задания**: `<номер>.practice.ipynb`
- **Тесты**: `<номер>.<тип>.ipynb` (`midterm`, `final`)

### 3. Требования к Pull Request
- Опишите суть изменений
- Убедитесь, что книга собирается без ошибок
- Все ноутбуки должны корректно открываться и выполняться в Google Colab

### 4. Процесс ревью
- PR проходит проверку maintainer-а
- После одобрения изменения сливаются в `main`
- GitHub Pages автоматически публикует обновленные материалы

## Локальная сборка и просмотр книги

### Сборка
```bash
# Полная сборка с принудительным выполнением всех ноутбуков
jupyter-book build --all materials
```

### Просмотр

Найдите `index.html` в `materials/_build/html` и откройте его в браузере.

Либо запустите локальный сервер:
```bash
# Запуск локального сервера
python -m http.server -d materials/_build/html
# Откройте http://localhost:8000
```
